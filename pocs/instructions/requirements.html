NVR & Tech Requirements

Everything the store must have ready before we install the Pi: the NVR, the cameras, the network and the room. Retail and F&B share the same list.

What is an NVR?

An NVR (Network Video Recorder) is the brain of the camera system: the one box that collects, stores and plays back the video from every camera in the store. Our Pi connects to the NVR to read the video, so the NVR has to be set up right before we install.

Dome and bullet cameras plug into the network, the NVR records them all, and XenReality's Pi reads from the NVR. One brain, many eyes.

Types of NVR

Stores can have different recorder types. Here's how to recognise each one, and what it means for us.

Standalone NVR

A dedicated recorder box for IP cameras. The most common setup and the one we prefer. Cameras and NVR talk over the network.

Works great

DVR

An older recorder for analog cameras (coax cables instead of network cables). No ONVIF, so we usually can't read from it directly.

Needs checking

Hybrid (XVR)

A recorder that accepts both analog and IP cameras. Fine for us as long as the cameras we need are IP and ONVIF is enabled.

Needs checking

PC-based VMS

Recording software running on a computer or server instead of a dedicated box. Works if it exposes ONVIF/RTSP streams on the local network.

Needs checking
Whatever the type: it must be ONVIF compatible, sit on the same Local Area Network as our Pi, and we need a username and password to access it.
